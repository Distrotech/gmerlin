GAVL API Documentation

1. Intoduction

1.1 Usage

Gavl can be used in two flavours: As a standalone library, which can be linked with executables for dynamic modules, and as a convenience library, which will statically linked with your application. In the last case, you should include gavl into the source distribution of your package. Switching between both modes is done by the --enable-library configure option. If you use this option, the library (libgavl.so.*) will be installed in <prefix>/lib, and the header file gavl.h in <prefix>/include. If you ommit the option, nothing will be installed.

You can integrate gavl in your autoconf/automake/libtool build tree. For this, you need to call the gavl configure script by adding it to AC_CONFIG_SUBDIRS in the top level configure.ac file. All files, which include gavl headers, need to be compiled with -I<gavl_directory>/include, where <gavl_directory> is the location of gavl relative to the sourcefile. To link your exectuable, add <gavl_cirectory>/gavl/libgavl.la to the *_LDADD variables.

All exported functions are defined in the header file gavl.h, so every sourcefile, which makes use of GAVL sould contain the line 

#include <gavl.h>

1.2 Using CPU specific features

For the conversion mode, the following constants are defined in gavl.h:

#define GAVL_ACCEL_C       (1<<0)
#define GAVL_ACCEL_C_HQ    (1<<1)
#define GAVL_ACCEL_MMX     (1<<2)
#define GAVL_ACCEL_MMXEXT  (1<<3)
#define GAVL_ACCEL_SSE     (1<<4)
#define GAVL_ACCEL_SSE2    (1<<5)
#define GAVL_ACCEL_3DNOW   (1<<6)

All routines are available in a Standard C version. C-High Quality functions aren't available for all routines, but they are always there, no matter on what platform gavl runs. Processor specific routines are selected acccording to runtime detection and compile time configuration. If you pass invalid flags here, they will silently be ignored. Normally, you will always pass GAVL_ACCEL_C and either GAVL_ACCEL_C_HQ (for highest quality) or the supported processor specific flags for higher speed requirements.

To get the supported accellerations (runtime detection) use:

int gavl_accel_supported();

Note, that GAVL_ACCEL_C and GAVL_ACCEL_C_HQ are never returned by this function, so you must OR these flags manually if you want to use them.

The audio- and videoconversions have an option structure, with an integer accel_flags. Set the flag to an ORed combination of the above flags to select the desired converison mode.
