GAVL

GAVL stands for Gmerlin Audio Video Library. It is an attempt to write an optimized, general purpose conversion library for uncompressed audio- and video data. The goal is to have a central location, where everyone can put his/her routines.

It currently supports only colorspace conversions and the allocation of video frames with pointers aligned at proper boundaries. In future versions, video scaling as well as audio resampling might also appear here.

It has the following features:

  • Extremely simple C API, which hides the internals of the conversion but gives a reasonable level of user control
  • Complete set of colorspace conversions, every colorspace can be converted to every other.
  • The use of function tables makes it possible to have several versions (C, MMX etc.) of a routine. Selection of the routines is based on  compile time configuration, runtime detection and user settable flags.
  • Heavy use of C macros creates 100s of conversion functions using only few, easily optimziable lines of code.
  • Integrated testsuite automatically creates timing comparisons for the different versions of all conversion routines
  • You need nothing but gcc to compile this. All other library dependencies were removed (only for the testsuite, you need the png library)
  • This library released is under the GPL license and cannot used by commercial applications, unfortunately :-)
Planned features are:
  • High quality mode with slower routines, which are fully standards compilant and suitable for high end applications
  • Maybe a few more colorspaces
  • Audio conversion routines: Mixing, resampling
  • Video scaling and deinterlacing
  • .....
This library is a side project of the gmerlin project. It replaces gmerlins own colorspace converter (which was too bad, really...). It is planned also to port gmerlins audio conversion engine (which isn't too bad) to gavl.
